Price Analysis and Monitoring

Problem

Xiaomi Indonesia utilizes two channels for product sales: the offline channel and the online channel. In the online channel, Xiaomi collaborates with various e-commerce platforms in Indonesia and maintains its own website. However, occasional pricing errors occur on some products listed on these e-commerce platforms. The online team (E-Commerce team) is responsible for verifying the accuracy of product prices across all e-commerce platforms where Xiaomi products are available. This meticulous price-checking process consumes a significant amount of time, primarily to ensure that all product prices offered by Xiaomi are correct and compliant with regulatory pricing standards.

Goal

The goal of this project is to streamline the process of verifying product prices on e-commerce platforms by implementing an automated system. This system will generate alerts and provide a dashboard for daily price monitoring. By doing so, the Online team can enhance their productivity by reallocating their time and resources to other tasks, ultimately reducing the need for manual price checks on a daily basis.

Duration

The duration of developing this project is around 8 - 9 weeks. The development consists of five phases:

  1. Gathering/Scrap price data automatically every day.
  2. Processing & Analyzing the data.
  3. Storing temporary data to compare the price.
  4. Create a dashboard using Power BI to monitor the price.
  5. Create an automatic alert by sending an email.

Tools

Steps

  1. Creating automation using Python, Selenium, and Power Automate to scrape data on all product prices from e-commerce platforms.
  2. Pre-processing the data by doing several steps such as handling missing values, duplicates, and outliers (data cleaning) using Python (Pandas).
  3. Stored temporary data to a database (MySQL) to make sure we have the last scraped data.
  4. Do some comparison between the latest scrap price data and the last scraped data that directly gets from the database with the help of Python.
  5. AFinal comparison data or clean data stored in Hive table that is available on Big Data Tools (known as Alpha).
  6. Analyze the clean data and create a dashboard to make it easier for the E-Commerce team to monitor the prices.

Result

  1. Reduce Time Consumption: Decrease the time spent on manual price checking across various e-commerce platforms on a daily basis.
  2. Increase Productivity: Boost the productivity of the E-Commerce team by 20% through the efficient allocation of their time and resources to other critical tasks.
  3. Accelerate Error Resolution: Expedite the resolution of pricing errors on specific e-commerce platforms by implementing an automated alert system. This system will continuously send emails to the responsible personnel at the e-commerce platform until the product prices are aligned with the regulatory standards.
